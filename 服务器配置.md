# 服务器配置文档

> **警告**：本文档包含敏感信息，请妥善保管，切勿提交到 GitHub 公有仓库！

---

## 1. 云服务器信息 (ECS)

| 配置项 | 值 |
|--------|-----|
| 服务器类型 | 阿里云轻量应用服务器 |
| 公网 IP | `YOUR_ECS_IP` |
| SSH 端口 | 22 |
| SSH 用户名 | root |
| SSH 认证方式 | 密钥对 |
| SSH 公钥 | `YOUR_SSH_PUBLIC_KEY` |
| SSH 私钥 | *(已提供给 CI/CD 系统)* |

---

## 2. 阿里云容器镜像服务 (ACR)

| 配置项 | 值 |
|--------|-----|
| ACR 名称 | 个人版 |
| ACR 地址 | `YOUR_ACR_REGISTRY` |
| 镜像仓库 | `spring-boot-hello` |
| 地域 | 华南-深圳 |
| 访问凭证 | 见下方 AccessKey |

---

## 3. 阿里云 AccessKey (用于 CI/CD 推送镜像)

| 配置项 | 值 |
|--------|-----|
| AccessKey ID | `YOUR_ACCESS_KEY_ID` |
| AccessKey Secret | `YOUR_ACCESS_KEY_SECRET` |

> **安全警告**：请在 GitHub Secrets 中配置以上信息，切勿明文写在代码中！
> 建议在阿里云控制台定期轮换 AccessKey。

---

## 4. GitHub Secrets 配置清单

在 GitHub 仓库的 `Settings → Secrets and variables → Actions` 中配置以下 Secrets：

| Secret 名称 | 对应值 | 说明 |
|------------|--------|------|
| `ACR_REGISTRY` | `YOUR_ACR_REGISTRY` | ACR 地址 |
| `ACR_USERNAME` | `YOUR_ACCESS_KEY_ID` | ACR 用户名 (同 AccessKey ID) |
| `ACR_PASSWORD` | `YOUR_ACCESS_KEY_SECRET` | ACR 密码 (同 AccessKey Secret) |
| `ECS_HOST` | `YOUR_ECS_IP` | ECS 公网 IP |
| `ECS_PORT` | `22` | SSH 端口 |
| `ECS_SSH_KEY` | *(SSH 私钥全文)* | 用于 SSH 连接到 ECS |

---

## 5. 服务器部署目录

| 配置项 | 值 |
|--------|-----|
| 应用部署路径 | `/opt/spring-boot-hello` |
| Docker 容器名称 | `spring-boot-hello` |
| 应用端口 | `8080` |
| 日志查看 | `docker logs -f spring-boot-hello` |

---

## 6. 开放的端口

| 端口 | 协议 | 用途 |
|------|------|------|
| 22 | TCP | SSH 远程连接 |
| 8080 | TCP | Spring Boot 应用端口 |
| 18789 | TCP | OpenClaw Gateway (如有) |

> **安全建议**：生产环境请使用安全组限制端口访问，只开放必要端口。

---

## 7. 重新部署命令 (手动)

如果 GitHub Actions 不可用，可以在服务器上手动执行：

```bash
cd /opt/spring-boot-hello
./deploy.sh
```

或手动拉取并启动：

```bash
docker pull YOUR_ACR_REGISTRY/spring-boot-hello:latest
docker stop spring-boot-hello 2>/dev/null || true
docker rm spring-boot-hello 2>/dev/null || true
docker run -d --name spring-boot-hello --restart unless-stopped -p 8080:8080 YOUR_ACR_REGISTRY/spring-boot-hello:latest
```
